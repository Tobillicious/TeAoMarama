# 🧠 SHARED KNOWLEDGE BASE

_"Whaowhia te kete mātauranga" - Fill the basket of knowledge_

## 🎯 PURPOSE

This shared knowledge base ensures all Cursor agents have access to the same information, maintaining consistency and enabling seamless coordination across the distributed network. Every agent that joins the system instantly gets up to speed with current context, protocols, and mission objectives.

---

## 📚 KNOWLEDGE CATEGORIES

### **1. Mission & Vision**

- **Educational Mission**: Serving 800,000+ akonga across Aotearoa
- **Cultural Foundation**: Te Ao Māori integration and tikanga compliance
- **Platform Goals**: Comprehensive, culturally-responsive educational resources
- **Success Metrics**: Performance, cultural safety, and educational impact

### **2. Technical Architecture**

- **System Overview**: React-based educational platform with Firebase backend
- **Key Components**: Resource browser, cultural learning modules, assessment systems
- **Performance Standards**: 99%+ uptime, <2s response times, accessibility compliance
- **Integration Points**: TeKeteAko API, cultural content systems, assessment frameworks

### **3. Cultural Protocols**

- **Tikanga Guidelines**: Proper cultural protocols and respect for Māori knowledge
- **Cultural Safety**: Validation processes and cultural flagging systems
- **Language Integration**: Te Reo Māori usage and pronunciation guidelines
- **Community Respect**: Honoring mana whenua and cultural heritage

### **4. Coordination Protocols**

- **Agent Hierarchy**: Supreme Overseer → Cultural Authority → Specialized Agents
- **Communication Channels**: Primary, status, decisions, and manifest systems
- **Conflict Resolution**: 3-level escalation ladder with cultural safety priority
- **Performance Monitoring**: Heartbeat systems and real-time status tracking

## 🔄 KNOWLEDGE SYNCHRONIZATION

### **Real-time Updates**

- **Shared Memory System**: JSON-based agent registry and status tracking
- **Heartbeat Integration**: Knowledge updates included in agent heartbeats
- **Change Propagation**: Automatic distribution of critical updates

### **Onboarding Process**

1. **Agent Registration**: New agents added to shared memory system
2. **Knowledge Transfer**: Complete knowledge base provided to new agent
3. **Protocol Training**: Cultural safety and coordination protocols
4. **Integration Testing**: Verify agent can access and update shared knowledge
5. **Active Monitoring**: Continuous health and performance tracking

### **Update Mechanisms**

- **Immediate Updates**: Critical changes pushed to all agents instantly

- **Change Notifications**: Agents notified of relevant knowledge updates
- **Conflict Resolution**: Handle conflicting knowledge updates appropriately

## 📋 DOCUMENTATION STANDARDS

### **File Naming Conventions**

- **Status Files**: `agent_sync_status.md`, `coordination_status_report.md`
- **Protocol Files**: `conflict_resolution_protocol.md`, `communication_protocol.md`
- **Decision Files**: `DECISIONS.md`, `migration_manifest.json`
- **Knowledge Files**: `shared_knowledge_base.md`, `cultural_protocols.md`

### **Content Structure**

```markdown
# Title

Brief description of purpose

## Section 1

Content with clear headings

### Subsection

Detailed information

---

## Section 2

More content
```

- **Last Updated**: ISO timestamp of last modification
- **Version**: Semantic versioning for tracking changes
- **Owner**: Agent responsible for maintaining the content
- **Review Status**: Cultural safety and technical validation status
